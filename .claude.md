## 1. Vue d'ensemble du projet

### Objectif
Créer un site web moderne, user-friendly et facile à maintenir pour l'association "Le Jardinier Sarthois" (section Aubigné-Racan / Vaas, Sarthe).

### Public cible
- Membres actuels et futurs de l'association
- Grand public intéressé par le jardinage
- Habitants de la région Sarthe

### Contraintes
- Budget : **GRATUIT** (hébergement et outils)
- Maintenance : **SIMPLE** et accessible aux non-développeurs
- Pas d'authentification (site public)
- Pas de base de données

---

## 2. Architecture technique

### Stack technique
```
HTML5 + Tailwind CSS + JavaScript Vanilla + Markdown
```

**Choix techniques :**
- **HTML5** : Structure sémantique et accessible
- **Tailwind CSS** : Framework CSS utilitaire moderne, rapide à implémenter
- **JavaScript Vanilla** : Interactions légères sans framework lourd
- **Fichiers Markdown** : Gestion du contenu (articles, événements)
- **Pas de base de données** : Simplicité maximale

**Hébergement (options gratuites) :**
- GitHub Pages (recommandé)
- Netlify
- Vercel
- Cloudflare Pages

## 3. Pages du site

### 1. Landing Page (Accueil) - `index.html`
- Hero section avec image de fond (légumes/jardin)
- Présentation de l'association (texte court et impactant)
- Appels à l'action (CTA) : "Nous rejoindre", "Voir les événements"
- Galerie miniature (3-4 photos)
- Prochains événements en aperçu

### 2. Actualités - `pages/actualites.html`
- Liste des articles (ordre chronologique inversé)
- Vignettes avec image, titre, date, extrait
- Pagination ou "Charger plus"
- Filtrage par tags/catégories (optionnel)

### 3. Agenda / Événements - `pages/agenda.html`
- Liste des événements à venir
- Format : Date, heure, lieu, description
- Affichage calendrier ou liste
- Événements passés en archive

### 4. Galerie Photos - `pages/galerie.html`
- Grille responsive d'images
- Catégories : Légumes, Fleurs, Événements, Équipe
- Lightbox pour agrandissement
- Lazy loading des images

### 5. Contact - `pages/contact.html`
- Informations de contact (email, téléphone)
- Formulaire de contact simple (optionnel)
- Adresse de l'association
- Carte (Google Maps embedded - optionnel)

### 6. À propos - `pages/a-propos.html`
- Histoire de l'association
- Membres du bureau
- Valeurs et objectifs
- Liens utiles

## 5. Règles de développement

### Conventions de nommage

**Fichiers :**
- HTML/CSS/JS : `kebab-case` (ex: `ma-page.html`)
- Images : `kebab-case` ou `snake_case` (ex: `photo-jardin.jpg`)
- Markdown : `YYYY-MM-DD-titre.md` pour articles datés

**Code :**
- Classes CSS : Tailwind utilities (pas de classes custom sauf nécessité)
- IDs HTML : `camelCase`
- Variables JS : `camelCase`
- Constantes JS : `UPPER_SNAKE_CASE`

### HTML - Sémantique obligatoire

```html
<!DOCTYPE html>
<html lang="fr">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta name="description" content="Description SEO">
    <title>Titre - Le Jardinier Sarthois</title>
</head>
<body>
    <header>
        <nav><!-- Navigation --></nav>
    </header>

    <main>
        <article><!-- Contenu --></article>
    </main>

    <footer><!-- Pied de page --></footer>
</body>
</html>
```

**Règles strictes :**
- Un seul `<h1>` par page
- Balises sémantiques : `<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<footer>`
- Attribut `alt` OBLIGATOIRE sur toutes les images
- `lang="fr"` sur `<html>`

### CSS - Approche Mobile-First

**Toujours développer mobile d'abord, puis ajouter les breakpoints :**
```css
/* Mobile par défaut */
.element { width: 100%; }

/* Tablette */
@media (min-width: 768px) {
    .element { width: 50%; }
}

/* Desktop */
@media (min-width: 1024px) {
    .element { width: 33%; }
}
```

**Avec Tailwind :**
```html
<div class="w-full md:w-1/2 lg:w-1/3">
    <!-- Contenu -->
</div>
```

### JavaScript - Principes

- **Vanilla JavaScript uniquement** (pas de jQuery, React, Vue)
- Code modulaire et fonctions pures
- Gestion des erreurs avec `try/catch`
- Commentaires JSDoc pour les fonctions
- Éviter les dépendances externes (sauf absolument nécessaire)

```javascript
/**
 * Charge les articles depuis les fichiers Markdown
 * @param {number} limit - Nombre maximum d'articles
 * @returns {Promise<Array>} Liste des articles
 */
async function loadArticles(limit = 10) {
    try {
        // Code...
    } catch (error) {
        console.error('Erreur:', error);
    }
}
```

## 7. Images - Règles strictes

### Optimisation OBLIGATOIRE

**Avant d'ajouter une image au site :**
1. Placer l'image dans `public/images/`
2. Exécuter : `python tools/optimize_images.py`
3. Vérifier que l'image fait < 500 Ko
4. Commit l'image optimisée

**Règles :**
- Format : JPEG pour photos, PNG pour logos, SVG pour icônes
- Dimensions max : 1920px
- Qualité JPEG : 85%
- Taille cible : < 500 Ko
- Noms descriptifs : `atelier-jardinage-2025.jpg`

### Usage dans HTML

```html
<!-- Image responsive avec lazy loading -->
<img
    src="/public/images/photo.jpg"
    alt="Description précise de l'image"
    loading="lazy"
    class="w-full h-auto rounded-lg shadow-md"
>
```

- [ ] **Attributs ARIA** : Quand nécessaire

### Exemple de navigation accessible

## 11. Git & Versioning

### Workflow

```bash
# Créer une branche feature
git checkout -b feature/nom-feature

# Commits atomiques fréquents
git add .
git commit -m "feat(page): ajouter page actualités"

# Merger sur main
git checkout main
git merge feature/nom-feature

# Tag pour versions
git tag v1.0.0
```

### Format des messages de commit

**Convention :** `type(scope): message`

**Types :**
- `feat` : Nouvelle fonctionnalité
- `fix` : Correction de bug
- `docs` : Documentation
- `style` : Formatage (pas de changement de code)
- `refactor` : Refactoring
- `perf` : Performance
- `test` : Tests
- `chore` : Tâches diverses

**Exemples :**
```
feat(gallery): ajouter lightbox
fix(nav): corriger menu mobile responsive
docs(readme): mettre à jour instructions
style(css): formater classes tailwind
refactor(js): simplifier loadArticles()
```
